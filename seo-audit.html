1. Critical Technical SEO Gaps Ship First

1.1 Missing sitemap.xml

No src/app/sitemap.ts exists. Google can't discover your URL set reliably. Ship a dynamic sitemap listing home, /about, /contact, /privacy-policy, /terms-of-service, and all six calculators with lastModified, changeFrequency: 'monthly', and priority (home + tax calc at 1.0, others 0.8).

1.2 Missing robots.txt

No src/app/robots.ts. Add one that allows all user agents, references the sitemap, and optionally blocks /api/ routes once you add them.

1.3 Broken internal link → /articles (404)

Header.tsx:44 points to /articles but the route doesn't exist. This wastes crawl budget and creates poor UX. Either build the hub or remove the link. The footer (Footer.tsx:28) simultaneously points to an external quickmoneytool.com/blog — pick one, and make blog/articles a first-class internal route.

1.4 Homepage has no canonical URL

src/app/layout.tsx sets metadataBase but no alternates.canonical. Every sub-page has a canonical — homepage does not. Add alternates: { canonical: '/' } to root metadata (or better, set it in the homepage route).

1.5 /calculators/ is not a real page

Breadcrumbs point to /#tools (an anchor, not a page). Google will not treat this as a breadcrumb node. Build a real /calculators/ index page: a hub with filters (by goal: "borrowing", "saving", "tax", "daily"), SEO copy, and interlinks to each calculator. This is also the site's most valuable internal link target.

1.6 Invalid SearchAction schema

schemaData.ts:17 points to /search?q={search_term_string} which doesn't exist. Google validates this — remove the potentialAction until you have a real site search, or you risk schema suppression.

1.7 Legacy .htaccess in /public

public/.htaccess is leftover from Hostinger/Apache hosting. On Vercel/Next.js it does nothing, but it's served as a static file and leaks X-Powered-By: Hostinger Horizons. Delete it.

1.8 Missing OG image + Twitter Card

Every page defines openGraph but no images array. No social previews = fewer clicks from social/referral traffic.

  • Add app/opengraph-image.png (1200×630) as root fallback
  • Per-calculator OG images (or dynamic opengraph-image.tsx using next/og)
  • Add twitter: { card: 'summary_large_image', ... } in each layout

1.9 No favicon pack

Only public/favicon.svg exists. Add icon.png (32×32), apple-icon.png (180×180), and a manifest.json via Next.js metadata icons convention.

1.10 <html lang> should be en-ZA

layout.tsx:39. You target South Africa exclusively; declare the locale explicitly. Add openGraph.locale: 'en_ZA' too.

2. Schema / Structured Data

2.1 Missing Organization schema

No publisher entity exists. Critical for E-E-A-T on YMYL (finance). Add to root layout:

{
  "@type": "Organization",
  "name": "Quick Money Tool",
  "url": "https://www.quickmoneytool.com",
  "logo": "https://www.quickmoneytool.com/logo.png",
  "sameAs": ["<twitter>", "<linkedin>", "<facebook>"],
  "contactPoint": {
    "@type": "ContactPoint",
    "email": "hello@quickmoneytool.com",
    "contactType": "customer support",
    "areaServed": "ZA"
  }
}

2.2 SoftwareApplication schema is thin

calculatorSchemaData.ts lacks aggregateRating, author, publisher, datePublished, dateModified, featureList, softwareVersion. Google surfaces rating stars and freshness signals from this.

2.3 No calculator-specific FAQ schema

Homepage has 4 FAQs. Every calculator page needs its own FAQ section with FAQPage schema attached. Currently /calculators/sars-income-tax-calculator has zero FAQs, missing massive long-tail traffic ("how is paye calculated", "when is tax return due in south africa").

2.4 Missing schemas per page type

  • /about → add AboutPage + Organization
  • /contact → add ContactPage with ContactPoint
  • Every calculator → add HowTo schema ("How to calculate your SARS income tax") with steps matching the form fields
  • /privacy-policy, /terms-of-service → WebPage with lastReviewed

2.5 Breadcrumb nodes pointing to anchors

"Calculators" → /#tools in every breadcrumb is invalid. Once a real /calculators/ page exists, wire the breadcrumb to it.

2.6 No BreadcrumbList on homepage

Homepage should itself declare a trivial BreadcrumbList with just "Home".

3. E-E-A-T — The Single Biggest Gap

Finance is YMYL (Your Money Your Life). Google holds it to the highest trust bar. Currently the site reads as anonymous — this is your #1 ranking ceiling.

3.1 No named authors / credentials

about/page.tsx:84 says "small, independent team of developers and finance enthusiasts" — no names, no qualifications. Competitors with named CFPs and tax practitioners will outrank you on query intent around SARS/retirement.

Fix:

  • Add author bios with photos, qualifications (SAICA, CFP®, FPI), LinkedIn
  • Add a "Reviewed by" line on every calculator page with a dated reviewer (e.g., "Reviewed by [Name], CA(SA), 16 April 2026")
  • Create /authors/[slug] pages with Person schema and knowsAbout

3.2 No "Last updated" dates

No calculator page shows when tax brackets / logic were last verified. Add a visible "Tax tables updated: March 2026" line near the H1 + dateModified schema.

3.3 No authoritative outbound citations

You reference SARS, NCR, SARB, Elizabeth Warren (50/30/20) — none are linked. Link to:

  • sars.gov.za (tax brackets/thresholds)
  • resbank.co.za (prime rate reference)
  • ncr.org.za (fee caps on personal loans — you quote R1,207.50 with no source)

Outbound links to .gov.za and major authorities lift trust meaningfully.

3.4 No trust social proof

No testimonials, usage counts, press mentions, or review widgets. Consider integrating a third-party review (Trustpilot) badge once you have volume.

3.5 Disclaimer placement

Disclaimer only lives on /about. Add a compact "Not financial advice — estimates only" disclaimer directly on every calculator results panel, linking to /about#disclaimer.

4. Content Strategy & Topical Authority

4.1 Long-form depth gap on calculators

Each calculator has ~300–500 words of educational content. Competitors (TaxTim, Old Mutual, 10X) have 1,500–3,000+ words per page. To be competitive for "sars tax calculator" (high volume, high commercial intent):

Add to the SARS page:

  • "Tax-free threshold explained" (~300w)
  • "How PAYE is deducted monthly" (~250w)
  • "Provisional tax vs PAYE" (~300w)
  • "Section 10(1)(o)(ii) foreign income exemption" (~200w)
  • "What is taxable: rental, side hustle, crypto" (~400w)
  • "When you don't need to submit a return" (~200w)
  • Worked example with screenshots
  • FAQ (8–12 questions)

Apply same scaling to all six calculators.

4.2 Blog/resource hub is missing

Header teases /articles but nothing ships. For topical authority in this space you need a content layer:

  • /articles/sars-tax-brackets-2026-2027 (direct keyword)
  • /articles/tax-free-savings-account-vs-retirement-annuity
  • /articles/vehicle-finance-vs-cash-purchase
  • /articles/50-30-20-budget-south-africa
  • /articles/interest-rates-explained
  • Glossary: /glossary/[term] for PAYE, UIF, SDL, TFSA, RA, Living Annuity (each 200–400w + DefinedTerm schema)

Each article interlinks to the most relevant calculator — closes the SEO → conversion loop.

4.3 Comparison content (high-intent long tail)

  • "Personal Loan vs Credit Card for Debt Consolidation"
  • "Balloon Payment: Pros and Cons for SA Car Buyers"
  • "RA vs Pension Fund vs Provident Fund"

These rank quickly and drive qualified traffic into calculators.

4.4 Missing cross-linking between calculators

A user on /calculators/vehicle-finance-calculator should see "Related: Personal Loan Calculator | Budget Calculator" at the bottom. Currently each page is an island — hurts dwell time and crawl equity distribution.

4.5 Homepage copy over-optimizes for brand

HeroSection.tsx:15 subheading lists 4 calculators in prose — fine. But consider H2s on homepage that hit modifier keywords: "Free SA tax calculator", "Retirement planning tools for South Africa", etc., so the homepage competes for broader head terms.

4.6 Currency converter content

Only ~300w, weak on local intent. Add sections: "Best time to exchange ZAR", "How banks price their forex", "Rand watch: what moves the Rand". Cross-link to SARS for travel allowance rules.

5. On-Page / Metadata

5.1 Meta descriptions are templated

All follow "Free X calculator for South Africa. Calculate Y..." — safe but uniform. Rewrite to include emotional hooks or specific CTA modifiers ("...in under 30 seconds", "updated 16 April 2026"). Aim for 150–158 chars.

5.2 Title tag optimization

Most titles good. Consider including the year in titles where freshness matters:

  • "SARS Income Tax Calculator 2026/2027 | Free SA Take-Home Pay Tool"
  • "Car Finance Calculator South Africa 2026 | Monthly Repayments"

5.3 H1 variants

Calculator H1s are plain ("SARS Income Tax Calculator"). Consider a slightly keyword-enriched variant: "SARS Income Tax Calculator for South Africa (2026/2027)" to match search intent fully.

5.4 Button anchor text is weak

ToolsSection.tsx:45 renders "Use SARS Calculator" by splitting tool.title on space — produces "Use Vehicle Calculator", "Use Currency Calculator" (wrong for "Currency Conversion"). Replace with descriptive CTAs like "Calculate my tax", "Estimate car repayments".

5.5 Hero CTA's internal-anchor design

Hero "View All Calculators" goes to #tools, not a real /calculators/ page. Once the hub exists, point primary CTAs there.

7. Performance / Core Web Vitals

7.1 Framer-motion weight on calculator pages

Every calculator page is 'use client' and imports framer-motion just for a hero fade-in (vehicle-finance-calculator/page.tsx:4). You already migrated the homepage hero to CSS transitions — continue that migration. Framer-motion is ~50KB gzipped; replacing these intros with animate-in fade-in Tailwind utilities (already in use) cuts meaningful JS.

7.2 All calculator pages force client-render

The whole route is a client component. Split the static SEO prose and hero into a server component wrapper, leave only the interactive form as a client boundary. This gives better TTFB and FCP.

7.3 No next/image usage anywhere

Zero image optimization infra. When you add author photos, OG images, diagrams, and article hero images, route them through next/image for AVIF/WebP + lazy loading.

7.4 Font loading

Inter with subsets: ['latin'] is good. Consider display: 'swap' explicitly and preloading only the weights used.

7.5 Third-party scripts

GA4 + AdSense are lazyOnload — good. Consider moving GA to afterInteractive if you need earlier page_view pings, or use Partytown for AdSense.

8. Accessibility (Indirect SEO)

  • Form labels present on contact page — good.
  • Header.tsx:14 nav items hidden md:inline-block means mobile users have no navigation at all — major UX and SEO signal issue (high bounce on mobile). Build a mobile menu.
  • Icon-only CTA buttons don't have visible text on all states — audit for aria-label.
  • Color contrast on text-muted-foreground against bg-accent/30 should be checked in WCAG — some areas look borderline.

9. Local / International SEO

  • Set <html lang="en-ZA">.
  • Add openGraph.locale: 'en_ZA' in root metadata.
  • Add Organization.address.addressCountry: 'ZA' in schema.
  • If you expand to other markets (Namibia, Botswana use similar tax logic), build hreflang strategy early.

10. Conversion / Engagement Features that Aid SEO

10.1 Shareable result permalinks

Encode calculator state in the URL (e.g., ?income=50000&age=under65). Enables:

  • Users sharing results (natural backlinks)
  • Ranking for long-tail calculation queries
  • Session depth / return visits

10.2 "Download as PDF" / "Email my results"

Drives email capture and creates a backlinkable artifact ("I used Quick Money Tool's SARS calculator — here's the PDF").

10.3 Comparison mode

e.g., on the vehicle calculator, compare two deals side-by-side. Very linkable content.

10.4 Embed widget

Offer a free HTML embed of your SARS calculator — blogs and forums will link back, earning DR. This is how NerdWallet/Bankrate win in the US.

12. Competitive Landscape Note

For "sars tax calculator" the SERP is dominated by TaxTim, Old Mutual, Sage, Fincheck, and SARS itself.

You will not outrank them on page one short term without:

  1. Named CA(SA) / CFP® authorship
  2. Deep content per calculator (1,500w+, FAQ, HowTo)
  3. Earned backlinks from SA fintech/media

The embeddable-widget + free-PDF strategy is your fastest path to DR growth — both TaxTim and Fincheck grew this way.
